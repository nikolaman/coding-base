# Еженедельный технический аудит

## Чеклист

Скопируй и пройди по пунктам. Результат — запись в `docs/SESSION-LOG.md` с датой.

### 1. Backlog и WIP
- [ ] `docs/BACKLOG.md` актуален
- [ ] Нет зависших `in_progress` > 1 недели
- [ ] WIP ≤ 2

### 2. Зависимости
- [ ] Устаревшие пакеты (`npm outdated` / `pip list --outdated` / аналог)
- [ ] Известные уязвимости (`npm audit` / `pip audit` / `cargo audit`)
- [ ] Неиспользуемые зависимости

### 3. Код и техдолг
- [ ] TODO/FIXME в коде — просмотрены, критичные в backlog
- [ ] Мёртвый код удалён или в backlog
- [ ] ADR актуальны (нет deprecated без замены)

### 4. Тесты и CI
- [ ] CI зелёный на main
- [ ] Критичная логика покрыта тестами
- [ ] Локально тесты проходят

### 5. Инфраструктура
- [ ] `docker-compose up` работает
- [ ] `.env.example` актуален (без секретов)
- [ ] CI workflow актуален

### 6. Бэкапы
- [ ] Бэкап БД настроен и не старше 7 дней
- [ ] Восстановление из бэкапа проверено (хотя бы раз в квартал)

### 7. Документация
- [ ] README актуален (запуск, env, архитектура)
- [ ] `docs/SESSION-LOG.md` — решения за неделю зафиксированы

### 8. Git
- [ ] Нет долгоживущих feature-веток (> 2 недель)
- [ ] main в актуальном состоянии

## Формат отчёта

```markdown
## Техаудит YYYY-MM-DD

### Критично (P0)
- ...

### Рекомендации (P1–P2)
- ...

### Идеи (P3)
- ...

### Следующий аудит
YYYY-MM-DD (+7 дней)
```

## Действия после аудита

1. Критичное → `docs/BACKLOG.md` с P0/P1
2. Отчёт → `docs/SESSION-LOG.md`
3. При необходимости — ADR для архитектурных находок
